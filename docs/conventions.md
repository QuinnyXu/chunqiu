# 春秋人物志 · 项目约定（conventions）

版本：v1.4（2026-07-18：私有层规则扩展为一切写作向列；校验器加公开表头护栏；私有文件改三列通用格式）
历史：v1.3（写作向备注列移出公开数据；presence 从严通例、后出叙事分层通例）
历史：v1.2（event_people 加 presence、events 加 sort_key；ID 退役规则）；v1.1（分类枚举 15 类增「相会」；来源六前缀；事件 ID 后缀口径）

所有协作者（含 AI 协作者）在改动数据或代码前必须先读本文件。

## 1. 目录结构与数据流

```
data/csv/          唯一数据源（source of truth），所有编辑发生在这里
data/incoming/     待合入的增量数据；复核通过后手工合入 data/csv/ 并删除
tools/             管线脚本（仅 Python 标准库）
site/              静态站根目录，可整体部署到 GitHub Pages
site/data/         生成的 JSON —— 生成物，禁止手改
site/assets/       图标（icons/）与地图底图（map/）
docs/              本文件与设计文档（design/）
private/           私有层（写作提示等，git 忽略，不入公开仓库）
```

数据流（单向）：

```
data/csv/*.csv  →  tools/csv_to_json.py  →  site/data/*.json + meta.json
```

- **JSON 禁止手改**：`site/data/` 下任何文件的改动只能来自重跑 `python tools/csv_to_json.py`。
- 每次改动 `data/csv/` 后的标准流程：`python tools/validate.py` 通过 → `python tools/csv_to_json.py` 重新生成 → 一起提交。

## 2. ID 规范

| 表 | 前缀/格式 | 示例 | 说明 |
|---|---|---|---|
| events | `E###`（可带一位大写字母后缀） | E010, E005A | **接续现有编号**（当前已用到 E020）；新事件尽量用纯 `E###`，字母后缀仅用于向既有编号之间插入事件 |
| people | `P_XXX`（大写拼音） | P_WENJIANG | |
| places | `L_XXX`（大写拼音） | L_LINZI | |
| sources | `Z###` 左传 / `S###` 史记 / `G###` 国语 / `A###` 考古 / `P###` 诗经 / `B###` 现代研究 | Z001, S002 | 六个前缀均合法，按来源性质归类 |
| passages | `Q###`（可带后缀） | Q004A | |
| background | `BKG###` | BKG001 | |
| archaeology | `ARC###` | ARC001 | |

ID 一经使用不得复用或改义；删除记录时其 ID 作废封存。

**ID 退役**：作废 ID 永不复用，也不得赋予新含义；引用它的历史提交以 git 历史为准。现有退役名单：**E005、E006**（已拆分为 E035/E036 与 E037–E039）。

## 3. 事件分类枚举（events.category）

共 **15** 个，图标一一对应（`site/assets/icons/` 下按同名维护）：

```
即位, 战争, 会盟, 相会, 婚嫁, 生育, 出奔, 弑杀, 薨卒, 丧葬, 外交, 内乱, 灾异, 礼俗, 其他
```

「相会」与「会盟」的区分：「会盟」指诸侯/国家层面的正式会、盟、朝聘；「相会」指个人之间的私会或非诸侯会盟性质的相见（主要用于文姜与齐襄公一线）。

- 每个事件只取一个分类；复合事件取对主线叙事最核心的一面，拿不准时在交付说明中列出待复核。
- `importance`：`1`=主线关键，`2`=重要，`3`=背景。

## 4. 地图投影规范

所有地图落点**必须按公式计算，不得手摆**。

- 覆盖范围：东经 110.0–122.0，北纬 32.0–38.5
- SVG `viewBox="0 0 1200 700"`
- 公式：

```
x = (lng - 110.0) / 12.0 * 1200
y = 700 - (lat - 32.0) / 6.5 * 700
```

校验样例（结果四舍五入到整数像素）：

| 地点 | lng, lat | x, y |
|---|---|---|
| 临淄 | 118.31, 36.83 | 831, 180 |
| 曲阜 | 116.99, 35.60 | 699, 312 |
| 新郑 | 113.73, 34.40 | 373, 442 |

`places.lat/lng` 若非空必须落在覆盖范围内（validate.py 强制校验）。坐标依据写入 `coord_basis`，确定性写入 `coord_certainty`。

## 5. reliability / certainty 三级标注

适用于 `events.reliability`、`places.certainty`、`places.coord_certainty`、`background.certainty`、`archaeology.certainty`：

- `high`：可作为骨架直接使用。
- `medium`：宜复核后使用。
- `low`：只适合作线索，不可直接当事实。

补充提醒（同旧库）：《诗经》条目是传统阐释和舆论层材料，不直接等同于编年史实；考古表目前是写作型摘要，学术级使用需补发掘报告出处。

## 6. CSV 编辑规约

- 编码 UTF-8，逗号分隔，首行表头；单元格内多值用 `;` 分隔（如 `source_ids`）。
- `year_bce` 为**负整数**表示公元前（前694 → `-694`），有效范围 [-800, -600]。
- 旧数据行的新增列允许留空；空值一律用空字符串，不写 `NULL`/`N/A`。
- 新增数据先进 `data/incoming/`，复核后合入 `data/csv/`。
- `event_people.presence`：枚举 `{亲至, 相关}`，允许暂空（**空视同亲至**）。亲至=此人当时身在事发地；相关=事件与其相关但本人不在场（如为其议婚、他人代行）。
- `events.sort_key`：可空整数；同一 `year_bce` 内按 sort_key 升序排列，按经传季月定值，建议以 10、20、30 间隔留插空。同一年内非空 sort_key 不得重复。

## 7. 私有层与叙事分层（v1.3）

- **私有层**：一切写作向列（历史上以 novel 开头命名的两个备注列，及后续任何写作向字段）不入公开数据，统一存 `private/writing_notes.csv`（通用三列 `table,row_id,note`；`private/` 整目录 git 忽略）。校验器对公开表头设有护栏：任何以 novel 开头的列名直接报错。增量包如有写作向备注，写在 CHANGES 附注区由合入者转存私有文件。
- **presence 从严通例**：凡"师行而君亲在无文"（某师伐某、某人灭某）、"使者代行"（使贼、使宛）、"称疾不往/不从"及追叙关联者，一律标「相关」；只有史文明书其人在事发地（会、盟、战、朝、奔、生卒于其地）方标「亲至」。宁严不滥。
- **后出叙事分层通例**：《史记》及更晚文献中戏剧化情节（射钩佯死、曹沫劫盟、彭生化豕之类），须在 summary 或摘录 quote_type 标明"后出叙事"层，不得与经传记事混写；科普展示面引用时保留分层标注。
