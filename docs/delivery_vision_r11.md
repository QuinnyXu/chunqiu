## 一、分享卡生成器（Xiangtao 反馈 5）

- **入口**：页脚新增分享行「生成分享卡 · 复制链接 ·（直接分享）」，与反馈行（GitHub Issues＋邮箱）相邻；页脚数据统计移入 `#footer-stats`（含 boot 失败提示），行为不变。
- **合成**（全前端 canvas，零依赖）：绢帛底＋青铜双线框＋上下回纹带（雷纹几何绘制，非贴图）＋站名（衬线逐字排距）＋九主角主题色签＋邀请语（`SHARE_COPY.invite`，即 copy_r8 N2 终审文）＋二维码（`assets/share/qr.png`，生绢衬块内整数倍缩放＋关闭平滑保扫码）＋域名文字。**两版尺寸**：1080×1440（3:4 竖图，微信/小红书）与 1080×1080（方图），对话框内胶囊切换。
- **中文排版细节**：canvas `letterSpacing` 兼容性不齐，一律逐字绘制控制字距；邀请语一行放不下时优先在「——」处分行（竖版实测正是此分行），一般兜底为标点不落行首的逐字换行。
- **操作**：预览 → 下载 PNG（toBlob）；「复制链接」（N4）写入主站 URL，成功 toast 用 N5「链接已复制，去分享给同好」；支持 Web Share API 的环境（含 Windows Edge 与多数手机浏览器）出现「直接分享」——对话框内优先携图（`canShare({files})`），不支持时退化为链接分享；页脚版分享链接＋邀请语。Esc/遮罩/✕ 关闭，对话框内 Tab 简易圈定，关闭后焦点还原。
- 字体用系统栈；canvas 导出为运行时功能，不属「预生成静态图」限制（任务书已明示）。

## 二、全站搜索（第 6 轮承诺）

- **位置**：页眉站名右侧一框（主导航之前）；≤680px 收纳为放大镜图标，点开占整行展开。
- **四组检索**（纯前端包含匹配，小写化＋去空格，字段间「|」隔断防跨字段误连；防抖 200ms）：
  - 人物：name＋alt_names＋姓/氏/名/字 → 主角直达其时间线；**非主角落其 ego 关系图**（详见「验收偏差上报」2）；
  - 地点：古名＋今地名 → 直达地图并高亮该点（人物语境优先，否则主角序中首个于此地有事件者；点在「活动范围」视野外时自动切全图）；
  - 事件：标题＋摘要＋事发地古今名（搜「临淄」应见发生于临淄之事）→ 所属主角时间线定位展开（朱砂描边一闪）；
  - 原文：quote_original＋modern_note（原文多繁体、白话注多简体，两者同索兼济繁简）→ 事件详情内定位到具体引文块；组内条目标注来源与叙事层（如「《史记·齐太公世家》（后出叙事）」）。
- **展示**：结果分组（人物/地点/事件/原文），每组先显 8 条＋「更多（还有 N 条）」组内展开；无结果给温和提示「库中未见此语——换一个词，或减一二字试试。」（copy_r8 无此项，自拟，报备）。
- **键盘**：combobox/listbox 语义；上下键循环选中（aria-activedescendant 同步）、Enter 直达（未选中即取首条）、Esc 关闭；「更多」也是可键选项。
- 事件归属主角的选取规则：当前人物语境优先，其次主角序中首个「亲至」者，再次任一关联主角（`protoForEvent`）；全部 99 事件、103 摘录均有主角落点（无主角关联者为 0，索引护栏仍在）。

## 已知问题 / 备忘

1. 搜索为内存包含匹配，繁简不互通：原文组靠 modern_note（简体白话注）兜住常用简体检索（「曹刿」如此命中），但纯繁体词（如「曹劌」）与纯简体词命中面不同；未引入繁简映射表（零依赖且避免手写大表），如成高频反馈可下轮由数据侧补 alt 文本或做小映射。
2. 「更多」展开后选中项定位到原「更多」所在序号（顺延首条新项），未特意滚到组头。
3. 移动端搜索面板下拉高度上限 60vh，超出组内滚动；无结果提示与桌面同款。
4. 页脚「直接分享」在桌面 Edge/Safari 也会出现（系统有原生分享面板即为真支持），非仅手机——与任务书「多为手机」描述一致，不做 UA 嗅探压制。
5. 分享卡对话框预览按 CSS 缩到 300px 宽显示，导出仍为全尺寸 1080 宽 PNG；下载文件名 `chunqiu-share-<宽>x<高>.png`。
6. r10 已知问题延续：人物语境为内存态（刷新丢失）；全景面包屑关系计数口径 112 条（数据事实）与并线 101 线并存；国别选项卡 aria-current 不随滚动联动。
