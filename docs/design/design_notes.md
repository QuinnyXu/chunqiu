## 1. 风格立场

**"简帛与青铜"，不是"仿古滤镜"。** 春秋气质来自三样东西：绢帛的底、墨与朱的书写、青铜的线。我们只取材质的色与线条的秩序，不做纹理贴图、不做描边金框、不做毛笔字特效。整体原则：**克制、清晰、留白**；装饰只出现在徽记与图标的线条本身。

### 十四主角主题色 —— 两级国色系（v1.6，裁定1）

**「国色定相、人色定阶、徽记定形」**：**国色**（家族）定色相，**人色**在家族内以深浅分阶（郑另借「青—绿」双相拉开距离），**徽记**形状恒定不变，作个人辨识第一锚点、不随色改。全站人物色（选人卡顶条与徽记、人物子导航、时间线加重、地图轨迹、关系 ego/全景节点、分享卡色签）一律取自 `site/styles.css` `:root` 的 `--p-<id>` 变量，**单点管理**；六家族色为 `--state-<国>`。`site/app.js` 于启动 `resolveProtoColors()` 读入填充，不再在代码内写死色值。

> 地图示意国色（`base_map.svg` 的 `layer-states` 淡色块）是另一套「示意疆域」层，与本节人物国色系**同名不同用、不同源**（如齐示意色为青灰、楚为苍绿），本轮不改；「同源同相」指人物色系内部与关系图阵营，地图重绘留待另议（见 delivery_vision_r14 验收偏差）。

## 3. 排版

- 标题/年号/引文：`"Songti SC","Noto Serif CJK SC","STSong","SimSun",serif`（衬线/宋体系，接近刻本气质）。
- 正文/界面：`system-ui,"PingFang SC","Microsoft YaHei",sans-serif`。
- 原文引文竖排？——**否**。竖排在 680px 下不可控，改为：引文用衬线、加左侧朱线、字距略宽（`letter-spacing:.05em`），示意简帛而不模仿简帛。
- 数字纪年双标：`前694 · 鲁桓公十八年`，公元前在前、鲁君纪年在后，中圆点分隔。

### 3.5 引文分层徽标（v1.5）

事件展开后的原文引文按 `passages.quote_type` 分层，各层给**专属色徽标＋同色左线＋极淡同色底**，一眼别层、风格统一（右上角胶囊徽标，`.q-layer`）。层与色的对应（呼应数据凡例「后出叙事分层通例」与来源可靠度）：

| 层（quote_type） | 徽标/左线色 | 含义 |
|---|---|---|
| 原文 | 朱砂 `#BC4433`（无徽标） | 《春秋》经传当代记事，骨架基准 |
| 言论 / 评论 | 淡墨 `#7A7166`（左线仍朱） | 经传之内的话语（言论）与断语（君子曰之评论），仍属经传层 |
| 后出叙事 | 淡墨 `#7A7166`，**左线灰虚线** | 《史记》等晚出戏剧化情节，虚线示「非当代记事」 |
| 诗歌 | 青绿 `#44766B` | 《诗经》舆论/阐释层（如秦风·黄鸟悼三良） |
| 经义异闻 | 乌梅紫 `#6E5A86` | 公羊/穀梁传注异说层（如公羊记崤之战），专属紫，与上列并列 |

样式统一（同一胶囊几何、同一淡底逻辑），仅换色相；未知 quote_type 走通用纯文本回退（不加徽标、不报错）。自验样本：崤之战 `E118` 公羊条 `quote_type=经义异闻`（紫徽标）、三良殉葬之黄鸟条 `诗歌`（青绿徽标）。

## 4. 图标语言（15 分类 + 9 徽记）

- 24×24 网格，线描 stroke 1.6，圆头圆角，`fill="none"`，`stroke="currentColor"`——颜色永远由使用处文字色/主题色决定。
- 意象取自器物与仪式，不画人脸、不画情节：即位=冕旒、战争=交戈、会盟=鼎、相会=月下二人、婚嫁=同心双环、生育=嘉禾、出奔=出门之矢、弑杀=倒悬之匕、薨卒=玉璧、丧葬=封土、外交=圭、内乱=城垣裂、灾异=星孛、礼俗=豆、其他=简牍。
- **会盟 vs 相会**：会盟是"器"（鼎，对称、庄重、直线多）；相会是"时"（残月与相向二人，弧线多、不对称、私密）。一眼可分。
- 徽记（48×48）走几何纹样抽象：文姜=涡纹、鲁桓公=圭璧、齐襄公=雷纹（方折回纹）、郑昭公=翎矢三叠、郑庄公=蟠虺双曲、鲁隐公=鱼纹（观鱼于棠）、鲁庄公=鼓纹、齐桓公=连环三盟、晋文公=济河之舟。圆框统一，纹样各异；同样 currentColor，由主题色着色。

## 5. 地图

- viewBox `0 0 1200 700`，投影严格按 conventions.md 公式；底图所有落点由脚本按同一公式生成（构建时校验：临淄→831,180、曲阜→699,312、新郑→373,442）。
- 图层顺序（自下而上）：海域 → 列国淡色底 → 水系 → 山地 → 古名标注 → 城邑锚点（空层，JS 填充）。
- 水系用青灰 `#7FA3A8`：河水取春秋故道（荥泽以下东北行，至今津沽入海），济水自荥泽分流东行经泺、临淄之北入海，另有汶、泗、颍、洧、淮。走向对照谭其骧图集的大势，线条做了平滑简化。
- 列国底色为低饱和差异色、透明度 0.35，只表"势力大致范围"，**不是考据疆界**（图例中注明）。
- 底图只标古名；今地名、坐标依据、certainty 全部进弹窗。
- 锚点样式：普通地点=空心小圆；当前人物相关=主题色实心；importance=1 事件地点在轨迹上加重。轨迹为主题色虚线按年序连线，端点标序号。

## 5.5 导航层级与 hash 结构（v1.2）

- **两级导航**：主导航只留全局项「选人 · 关系 · 资料库 · 关于 · 反馈」；选定人物后头部下方出现**人物子导航**「〔徽记+人物名〕· 时间线 | 地图 | 关系 | ✕ 换人」，次级样式（绢帛底、虚线下边、胶囊小按钮、主题色高亮），视觉从属于主导航。站名＋徽记整体为回选人页链接。
- **hash 新结构**：`#/`（选人·地图模式）、`#/?home=list`（选人·列表模式，v1.4）、`#/p/<PID>/timeline|map|relations`（人物视图，relations=ego）、`#/relations`（全景）、`#/library[/<tab>][?q=…]`、`#/about`。旧格式 `#person=X&view=…` 由 `legacyToNewHash` 就地 `history.replaceState` 改写，外部旧链接永不失效。
- **人物语境**（`personCtx`）：进过人物视图后记住；逛资料库/关于时子导航仍在、主题色保持，「✕ 换人」清除。全局视图的 hash 不携带人物（分享资料库/关于链接是干净的）。
- **关系视图语义**：主导航「关系」已选人→该人物 ego 图，未选人→56 人全景（带「仅主角边」过滤器）；首页「全景关系图谱」入口恒指全景。
- **关系并线**：同一对人物的多条关系合并为一条边，取类型序（REL_COLORS 键序）首条定边色；多重关系在边中点加**数字徽记**（生绢底小圆＋计数，点击同边）；虚线=该对关系可靠度皆中/低（规则详注入关系卡）。关系卡按类型排序列出全部关系（rel_label、可靠度、依据）。
- **地点抽屉**：全屏态一律走底部抽屉；>680px 收为限宽浮卡（`min(520px, 100vw−2rem)` 居中、圆角 12px、底距 18px、浮起投影），≤680px 保持整宽贴底。全屏地图的指针捕获延迟到确认拖移/捏合才设置，保证桌面鼠标 click 可达锚点。

## 5.6 全站搜索（v1.3）

- **位置**：页眉站名右侧、主导航之前；680px 下收纳为放大镜图标，点开占整行展开输入框。
- **口径**：纯前端包含匹配（小写化＋去空格），字段间以「|」隔断防跨字段误连；输入防抖 200ms。四组：人物（name/alt_names/姓/氏/名/字）、地点（古名＋今地名）、事件（标题＋摘要＋事发地古今名——搜「临淄」应见发生于临淄之事）、原文（quote_original＋modern_note，白话注多为简体，兼济繁简检索）。
- **直达语义**：人物（主角）→其时间线；人物（非主角）→其 ego 关系图（hash 落 `#/relations`，同「以人为中心」游走态）；地点→地图并高亮（人物语境优先，否则主角序中首个于此地有事件者；活动范围外自动切全图视野）；事件→所属主角时间线定位展开（朱砂描边一闪 `spotlight`）；原文→事件详情内定位到引文块。
- **展示**：每组先显 8 条＋「更多（还有 N 条）」组内展开；无结果一句温和提示（`SEARCH_EMPTY`，copy_r8 无此项故自拟）。
- **键盘**：combobox/listbox 语义，上下键循环选中（aria-activedescendant）、Enter 直达（未选中即首条）、Esc 关闭。

## 5.7 分享卡与底部分享行（v1.3）

- **构图**（canvas 运行时合成，零依赖）：绢帛底＋青铜双线框＋上下回纹带（雷纹钥匙纹，几何绘制，非贴图）＋站名（衬线逐字排距）＋九主角主题色签一行（取 PROTAGONISTS 配置）＋邀请语（SHARE_COPY.invite，一行放不下优先在「——」处分行）＋二维码（生绢衬块、整数倍缩放关平滑保扫码）＋域名。两版：1080×1440（3:4 竖图，微信/小红书）与 1080×1080（方图）。
- **字体**：canvas 用系统栈（SHARE_SERIF/SHARE_SANS），letterSpacing 兼容性不齐故逐字绘制。
- **入口**：页脚「生成分享卡 · 复制链接 ·（直接分享）」与反馈行（Issues＋邮箱）相邻；「直接分享」仅在支持 Web Share API 的环境出现，优先携图（canShare files），退化为链接分享。复制链接写入主站 URL，成功 toast 用 N5。
- **对话框**：尺寸切换胶囊、预览、下载 PNG（toBlob）；Esc/遮罩/✕ 关闭，Tab 简易圈定。

## 6. 三屏结构与交互

- 屏1 选人（#/）：v1.4 起默认地图导航（见 5.8），列表模式保留如下：按 state 首国分区（v1.1），组头=国名大字+一句话气质注（界面文案，配置于 app.js `STATE_EPITHET`）；顶部国别选项卡（全部/齐/鲁/郑/晋…随分区自动生成）滚动定位到分区，sticky 吸顶，680px 下横滑。卡=徽记+姓名+姓名行小字（姓/氏/名/字压缩形式）+生卒+一句话简介，主题色作卡顶条与徽记色；跨国人物（state 含「/」）在姓名旁标流向（如文姜「齐→鲁」）。数据缺席的主角显示"资料整理中"占位卡（不可进入）。新国加入只增分区，布局自动扩展。
- 人物子导航（v1.4 升格）：人物名主题色衬线加粗、字号升一档（1.08rem），徽记 24px；未激活视图标签用正文玄墨（此前淡墨对比不足，Xiangtao 反馈 3）；激活态维持主题色深底白字；子导航在场时 main 顶部留白加 0.5rem 拉开层次。
- 屏2 时间线（#/p/<PID>/timeline）：顶部年份刻度尺（min→max，十年一格），主体为 `<details>` 纵列：图标+双标年份+标题；importance=1 加粗左线并放大图标；展开显示摘要、地点、可靠性、原文引文（passages，附出处）。
- 屏3 地图（#/p/<PID>/map）：底图+该人物轨迹；点击/回车锚点开弹层（古名、今地、坐标依据、certainty、该人物相关事件）；无坐标地点列在侧栏"未定位地点"；"按时间播放"按钮做逐段轨迹动画（可暂停）。
- URL hash 全状态可分享；浏览器前进后退可用。
- 键盘：卡片与锚点均可 Tab 聚焦、Enter 激活；`<details>` 原生可达；焦点环用暖赭双线。
