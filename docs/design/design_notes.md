# 春秋人物志 · 视觉设计定稿（design_notes v1.2）

作者：Vision。适用范围：site/ 各屏、site/assets/icons/、site/assets/map/base_map.svg。
版本：v1.2（2026-07-18：导航两级层级与人物子导航；hash 新结构；关系并线与数字徽记；桌面全屏抽屉浮卡）。
历史：v1.1（晋文公主题色与徽记定稿；选人页按国分组与国别选项卡；姓名行显示样式）；v1.0。

## 1. 风格立场

**"简帛与青铜"，不是"仿古滤镜"。** 春秋气质来自三样东西：绢帛的底、墨与朱的书写、青铜的线。我们只取材质的色与线条的秩序，不做纹理贴图、不做描边金框、不做毛笔字特效。整体原则：**克制、清晰、留白**；装饰只出现在徽记与图标的线条本身。

## 3. 排版

- 标题/年号/引文：`"Songti SC","Noto Serif CJK SC","STSong","SimSun",serif`（衬线/宋体系，接近刻本气质）。
- 正文/界面：`system-ui,"PingFang SC","Microsoft YaHei",sans-serif`。
- 原文引文竖排？——**否**。竖排在 680px 下不可控，改为：引文用衬线、加左侧朱线、字距略宽（`letter-spacing:.05em`），示意简帛而不模仿简帛。
- 数字纪年双标：`前694 · 鲁桓公十八年`，公元前在前、鲁君纪年在后，中圆点分隔。

## 4. 图标语言（15 分类 + 9 徽记）

- 24×24 网格，线描 stroke 1.6，圆头圆角，`fill="none"`，`stroke="currentColor"`——颜色永远由使用处文字色/主题色决定。
- 意象取自器物与仪式，不画人脸、不画情节：即位=冕旒、战争=交戈、会盟=鼎、相会=月下二人、婚嫁=同心双环、生育=嘉禾、出奔=出门之矢、弑杀=倒悬之匕、薨卒=玉璧、丧葬=封土、外交=圭、内乱=城垣裂、灾异=星孛、礼俗=豆、其他=简牍。
- **会盟 vs 相会**：会盟是"器"（鼎，对称、庄重、直线多）；相会是"时"（残月与相向二人，弧线多、不对称、私密）。一眼可分。
- 徽记（48×48）走几何纹样抽象：文姜=涡纹、鲁桓公=圭璧、齐襄公=雷纹（方折回纹）、郑昭公=翎矢三叠、郑庄公=蟠虺双曲、鲁隐公=鱼纹（观鱼于棠）、鲁庄公=鼓纹、齐桓公=连环三盟、晋文公=济河之舟。圆框统一，纹样各异；同样 currentColor，由主题色着色。

## 5. 地图

- viewBox `0 0 1200 700`，投影严格按 conventions.md 公式；底图所有落点由脚本按同一公式生成（构建时校验：临淄→831,180、曲阜→699,312、新郑→373,442）。
- 图层顺序（自下而上）：海域 → 列国淡色底 → 水系 → 山地 → 古名标注 → 城邑锚点（空层，JS 填充）。
- 水系用青灰 `#7FA3A8`：河水取春秋故道（荥泽以下东北行，至今津沽入海），济水自荥泽分流东行经泺、临淄之北入海，另有汶、泗、颍、洧、淮。走向对照谭其骧图集的大势，线条做了平滑简化。
- 列国底色为低饱和差异色、透明度 0.35，只表"势力大致范围"，**不是考据疆界**（图例中注明）。
- 底图只标古名；今地名、坐标依据、certainty 全部进弹窗。
- 锚点样式：普通地点=空心小圆；当前人物相关=主题色实心；importance=1 事件地点在轨迹上加重。轨迹为主题色虚线按年序连线，端点标序号。

## 5.5 导航层级与 hash 结构（v1.2）

- **两级导航**：主导航只留全局项「选人 · 关系 · 资料库 · 关于 · 反馈」；选定人物后头部下方出现**人物子导航**「〔徽记+人物名〕· 时间线 | 地图 | 关系 | ✕ 换人」，次级样式（绢帛底、虚线下边、胶囊小按钮、主题色高亮），视觉从属于主导航。站名＋徽记整体为回选人页链接。
- **hash 新结构**：`#/`（选人）、`#/p/<PID>/timeline|map|relations`（人物视图，relations=ego）、`#/relations`（全景）、`#/library[/<tab>][?q=…]`、`#/about`。旧格式 `#person=X&view=…` 由 `legacyToNewHash` 就地 `history.replaceState` 改写，外部旧链接永不失效。
- **人物语境**（`personCtx`）：进过人物视图后记住；逛资料库/关于时子导航仍在、主题色保持，「✕ 换人」清除。全局视图的 hash 不携带人物（分享资料库/关于链接是干净的）。
- **关系视图语义**：主导航「关系」已选人→该人物 ego 图，未选人→56 人全景（带「仅主角边」过滤器）；首页「全景关系图谱」入口恒指全景。
- **关系并线**：同一对人物的多条关系合并为一条边，取类型序（REL_COLORS 键序）首条定边色；多重关系在边中点加**数字徽记**（生绢底小圆＋计数，点击同边）；虚线=该对关系可靠度皆中/低（规则详注入关系卡）。关系卡按类型排序列出全部关系（rel_label、可靠度、依据）。
- **地点抽屉**：全屏态一律走底部抽屉；>680px 收为限宽浮卡（`min(520px, 100vw−2rem)` 居中、圆角 12px、底距 18px、浮起投影），≤680px 保持整宽贴底。全屏地图的指针捕获延迟到确认拖移/捏合才设置，保证桌面鼠标 click 可达锚点。

## 6. 三屏结构与交互

- 屏1 选人（#/）：按 state 首国分区（v1.1），组头=国名大字+一句话气质注（界面文案，配置于 app.js `STATE_EPITHET`）；顶部国别选项卡（全部/齐/鲁/郑/晋…随分区自动生成）滚动定位到分区，sticky 吸顶，680px 下横滑。卡=徽记+姓名+姓名行小字（姓/氏/名/字压缩形式）+生卒+一句话简介，主题色作卡顶条与徽记色；跨国人物（state 含「/」）在姓名旁标流向（如文姜「齐→鲁」）。数据缺席的主角显示"资料整理中"占位卡（不可进入）。新国加入只增分区，布局自动扩展。
- 屏2 时间线（#/p/<PID>/timeline）：顶部年份刻度尺（min→max，十年一格），主体为 `<details>` 纵列：图标+双标年份+标题；importance=1 加粗左线并放大图标；展开显示摘要、地点、可靠性、原文引文（passages，附出处）。
- 屏3 地图（#/p/<PID>/map）：底图+该人物轨迹；点击/回车锚点开弹层（古名、今地、坐标依据、certainty、该人物相关事件）；无坐标地点列在侧栏"未定位地点"；"按时间播放"按钮做逐段轨迹动画（可暂停）。
- URL hash 全状态可分享；浏览器前进后退可用。
- 键盘：卡片与锚点均可 Tab 聚焦、Enter 激活；`<details>` 原生可达；焦点环用暖赭双线。

## 7. 明确不做

- 不引外部字体/库/CDN；不做构建步骤。
- 不做暗色主题（绢帛底即品牌，本轮不出夜间版）。
- 不在代码中手写任何史实文案，一切叙事文本来自 site/data/*.json。
